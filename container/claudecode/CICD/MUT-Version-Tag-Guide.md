# MUTのバージョンタグ管理

## バージョン付与ルール

**基本設計書による方針:**
- バージョン付与はメジャーとマイナーのみで基本は２つだけで運用
- リビジョンは任意のタイミングとして内部用とする

**バージョン形式:**
```
運用版: メジャー.マイナー (例: 0.1, 0.2, 1.0)
内部用: メジャー.マイナー.リビジョン (例: 0.1.1, 0.2.3)
```

## バージョン区切り

| バージョン | 内容 |
|-----------|------|
| 0.1.0 | javaのCI化 |
| 0.2.0 | javascriptのCI化 |
| 0.3.0 | 基本ルートの登録 |
| 0.4.0 | 実行環境へのデプロイと起動確認用 |
| 0.5.0 | ITむけにArgoCDまで含めたCICDの完全版 |
| 1.0.0 | 最終納品向けCIクリア版 |

## CI実行方針

**0.4.0まで:**
- CIは定期的にうごかす
- Gitへのプッシュトリガーでは起動させない
- 各自jacocoとsonarqubeはローカルで実行する
- 定期的に実行されるCIのcourtsを見てもよい

**0.5.0以降:**
- プッシュトリガー有効化
- 完全自動CI/CD実行

## ローカル品質チェック実行例

**JaCoCo実行:**
```bash
mvn clean test jacoco:report
```

**SonarQube実行:**
```bash
mvn clean verify sonar:sonar -Dsonar.projectKey=courts -Dsonar.projectName="courts"
```
